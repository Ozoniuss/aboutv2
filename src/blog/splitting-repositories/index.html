Splitting a big repository into smaller ones

Let me give you a quick teaser to think about before reading this article: imagine you have a huge monorepo, where you keep two different projects that don't have anything in common (e.g. no commit changed files in both projects). You want to create two different repos for each of the two projects, whilist transfering all commits that contributed to each individual project in their new repo. How do you approach this? (SPOILER: that's what this article is about)

I have a big Algorithms repository where I keep solutions for a bunch of coding problems. While this was fine for the beginning, the more problems I added, the less this monolith structure made sense. For example, I have a Leetcode folder and an Advent of Code folder, which not only contain solutions but also various helpers for their specific type of problems. At least those two folders I wanted to be separate repos, to have the problems and helpers live independently.

However, I also wanted to keep their commit history, and not just make in each of the new repos a huge commit titled "add all existing problems" or some shit. So I knew right away I was in for some heavy git rebase work. Luckily, the Leetcode commits have nothing to do with the Advent of Code ones, so at list there was no need to break any existing commits.
